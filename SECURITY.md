# セキュリティポリシー

SyncPulse Platform に関する脆弱性のご報告に感謝します。本ドキュメントでは報告の流れと対応方針をまとめます。

## サポート対象

| ブランチ | 状態 |
| --- | --- |
| `main` | サポート対象（最新の修正を反映） |
| その他 | サポート対象外 |

リリースタグを付与する運用に移行した場合は、本セクションを最新 N 系に絞って更新します。

## 報告フロー

公開 Issue では報告しないでください。第三者が悪用可能な情報を晒すリスクがあるためです。

GitHub の Private vulnerability reporting を利用してください:

1. リポジトリの **Security** タブ → **Report a vulnerability** を開く
2. 内容を入力して送信

リンク: https://github.com/mohadayo/syncpulse-platform/security/advisories/new

GitHub の私信機能が利用できない場合のみ、リポジトリオーナーへ直接連絡してください（GitHub プロフィール記載の連絡先）。

## 報告に含めて欲しい情報

| 項目 | 概要 |
| --- | --- |
| 影響範囲 | どのサービス・コンポーネントが影響するか |
| 重大度 | CVSS 等の指標があれば添付 |
| 再現手順 | コマンドラインや HTTP リクエストの最小再現例 |
| 想定される影響 | 情報漏洩 / RCE / DoS など |
| PoC | 任意。既存サービスを破壊しない範囲で |
| 推奨修正案 | 任意。コードレベルの修正方針があれば |
| 報告者 | 公開時のクレジット可否、希望する表記 |

## 応答 SLA

| マイルストーン | 目安 |
| --- | --- |
| 初動応答 | 72 時間以内 |
| 再現確認 | 7 日以内 |
| 修正方針共有 | 14 日以内 |
| 修正リリース | 重大度に応じて 30〜90 日以内 |

進捗は GitHub Security Advisory のスレッド上で共有します。

## 公開ディスクロージャ方針

- 修正リリース後にアドバイザリを公開し、CVE が割り当てられている場合は併記します
- 報告者の希望に応じてクレジットを記載します（匿名希望も尊重）
- 修正前の詳細情報を一般公開する場合は、メンテナと事前に調整してください

## スコープ外

以下は本ポリシーのスコープ外です:

- 第三者依存パッケージそのものの脆弱性（該当パッケージの upstream へ報告ください）
- Docker / クラウドプロバイダ等、本リポジトリで管理しないインフラ層
- 自分で改造したフォーク・派生物
- ソーシャルエンジニアリング、物理攻撃、DoS 攻撃の試行
